# MedHora - Documentacao

Esta pasta organiza a documentacao tecnica do MedHora por tema.

## Indice

| Documento | Conteudo |
| --- | --- |
| `REQUISITOS.md` | Requisitos funcionais, nao funcionais, seguranca, privacidade e pendencias |
| `DIAGRAMAS.md` | Diagramas Mermaid de arquitetura, componentes, autenticacao e modelo de dados |
| `ARQUITETURA.md` | Visao arquitetural, camadas, modelo de dados, operacao e roadmap |
| `PADROES_DO_PROJETO.md` | Padroes de projeto e organizacao usados no frontend, backend e banco |
| `COMPATIBILIDADE_DO_SISTEMA_COM_A_DOCUMENTACAO.md` | Compatibilidade entre documentacao e sistema implementado |

## Leitura Recomendada

1. Comece por `REQUISITOS.md` para entender o que o sistema deve cumprir.
2. Leia `ARQUITETURA.md` para entender como o sistema esta organizado.
3. Use `DIAGRAMAS.md` para visualizar os fluxos principais.
4. Consulte `PADROES_DO_PROJETO.md` antes de criar novas estruturas ou refatorar modulos.
5. Revise `COMPATIBILIDADE_DO_SISTEMA_COM_A_DOCUMENTACAO.md` para conferir o estado implementado.
